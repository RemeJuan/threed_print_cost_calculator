# Crashlytics Issue - 89453e01c8a4c3e09d6f0bb393933f72

Date: 2026-05-26

## Summary

Fatal Android crash in `PricingCalculator._toCents`.

Crash signature:
- `FormatException: Positive input exceeds the limit of integer`
- offending value: `19369081277395030400`
- first/last seen in `2.10.0 (204)`

## Root Cause

`lib/calculator/helpers/pricing_calculator.dart` converts decimal values to cent strings and then calls `int.parse`. Oversized numeric input can exceed Dart int range and crash before analytics logging completes.

`_toCents` is used for:
- `baseCost`
- `markupPercent`
- `setupFee`

Rounding mode is not the root trigger. It may make pricing visible in the UI, but the overflow happens before rounding is applied.

## Fix Plan

1. Make pricing normalization overflow-safe.
2. Add regression tests for huge numeric inputs.
3. Add input bounds or validation on pricing-related fields.
4. Sanitize hydrated settings/history values before they reach pricing.
