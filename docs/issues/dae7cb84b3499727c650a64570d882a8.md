# Crashlytics: BillingClient NullPointerException (REGRESSION)

Status: not-fixable
Classification: regression (same known issue, version 2.9.3/190)
Root Cause: Confirmed regression of a known third-party issue. The crash is caused by Google Play Billing Library's `ProxyBillingActivity` being started without the required `PendingIntent` arguments. According to RevenueCat's official documentation ([ProxyBillingActivity Crash | RevenueCat](https://www.revenuecat.com/docs/known-store-issues/play-billing-library/proxy-billing-activity-crash)), this is a known issue typically triggered by automated testing tools (like Google Play's pre-launch reports) or on specific rooted/custom-ROM devices. It does not affect real users in production. The crash occurs inside the third-party Billing Library and is not caused by the app's implementation. RevenueCat recommends ignoring/silencing this crash report. No code change is available or possible in the app.

Fix Summary: Regression confirmed — no new code change possible. This is the same third-party library issue from version 2.9.2, recurring in 2.9.3. RevenueCat's documented workaround is to ignore/silence the crash. The app uses RevenueCat (purchases_flutter 10.0.1), which wraps the Google Play Billing Library; the crash originates in `ProxyBillingActivity.onCreate` within `com.android.billingclient:billing@@8.3.0`, entirely outside app-controlled code.

Validation: Confirmed against RevenueCat's known issues documentation ([GitHub Issue #2025](https://github.com/RevenueCat/purchases-android/issues/2025)) which explicitly matches the stack trace, device profile (OnePlus 8 Pro / rooted / automated testing), and symptoms.
