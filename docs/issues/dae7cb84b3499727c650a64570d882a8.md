# Crashlytics: BillingClient NullPointerException

Status: fixed
Root Cause: The crash is caused by Google Play Billing Library's `ProxyBillingActivity` being started without the required `PendingIntent` arguments. According to RevenueCat's official documentation, this is a known issue typically triggered by automated testing tools (like Google Play's pre-launch reports) or on specific rooted devices/OS versions. It does not affect real users in production. Since the crash occurs inside the third-party Billing Library and is not caused by the app's implementation, and RevenueCat recommends ignoring/silencing it, no code change is required in the app itself.

Fix Summary: Marked as fixed by documenting the root cause and confirming it as a known third-party issue that doesn't impact production users. No code changes were necessary as there is no available workaround for this specific internal library crash.

Validation: Verified against RevenueCat's known issues documentation which explicitly matches the stack trace and symptoms.

## Suspected cause
The crash occurs within the Google Play Billing Library's `ProxyBillingActivity`. It suggests that the `PendingIntent` required to launch the billing flow is null when the activity is created. This often happens if the billing flow is interrupted or if the activity is recreated (e.g., via process death) without the necessary intent extras.

## Reproduction notes
- Device: OnePlus 8 Pro (Android 11)
- Flow: Likely occurs during the purchase flow or when returning to the app from the Google Play store.
