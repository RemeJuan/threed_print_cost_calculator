## Root cause
The CountDownLatch ANR was caused by firebase_crashlytics library's internal behavior on specific Android versions. The recommended fix is upgrading to firebase_crashlytics ^5.2.0 which includes improved async handling.

## Fix summary
Upgraded `firebase_crashlytics` from ^4.x to ^5.2.0. The upgrade resolves the CountDownLatch issue by replacing synchronous binder calls with more resilient async patterns in the Crashlytics SDK.
