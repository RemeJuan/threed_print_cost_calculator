## Suspected cause
- `AppPage` showed the whats-new bottom sheet as soon as `currentAnnouncementProvider` resolved, even if `AppPage` was no longer the active route.
- If a secondary route/activity was already foregrounded (`LicenseActivity` in breadcrumbs), stale `AppPage` context could still push the non-dismissible modal.
- Most plausible ANR path: Flutter surface recreation during route/activity transition while the whats-new modal was being presented.

## Root cause
- The announcement async resolution could race with a route transition. A non-dismissible modal presented from stale `AppPage` context during Flutter surface recreation causes an ANR on Android.
- Fix (da15978): latch moved after double `canShowAppPageModalSheet` check, with `context.mounted` guard inside post-frame callback.

## Root cause
- The previous guard was not sufficient for the regression path in 2.10.0 because `whatsNewShown` was still being set before the route/lifecycle check completed, so an async announcement could still race with a route transition and present the sheet from a stale context.
- Third regression root cause: regression test was removed during main branch feature work (commit 1f87a4b), leaving no automated guard against reintroducing the race condition.

## Fix summary
- da15978: Moved the `whatsNewShown` latch to after the route/lifecycle guard and post-frame callback. Added a defensive mounted check and kept the sheet suppressed when `AppPage` is not current or the app is not resumed.
- v4: Restoring the regression test that was removed during main branch cleanup.

## Risk assessment
- Low. Change only narrows when the announcement may appear.
- Trade-off: if announcement resolves while another route/activity is active, it is skipped for that session and can appear on a later launch instead.
